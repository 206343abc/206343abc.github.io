Project Assessment Focus: Excellence in Delivery

This section details how the project structure and content directly satisfy the grading requirements, emphasizing **Creativity**, **Relevance**, and **Functionality**.

Detailed Deliverable Breakdown

  • Creativity & Innovation:
    We chose a dynamic, multi-page HTML/CSS/JS hub structure over a static document. The Interactive Checklist (Local Storage persistence) and the integration of specific, peer-relevant coping mechanisms (e.g., Sensory Reset) demonstrate innovative tool application.
  • Relevance & Impact:
    Content is curated based on common student stressors (exam anxiety, social overload, sleep debt). Tools are categorized by *immediate need* vs. *long-term cultivation*, ensuring maximum practical impact for daily student life.
  • Collaboration & Structure:
    The three files (`index.html`, `journal.html`, `selfcare.html`) clearly represent distinct, modular components, proving effective task division and integration. Consistent styling across all 3 pages ensures a unified user experience.
  • Presentation & Aesthetics:
    Utilizes modern, professional typography (`Poppins` & `Inter`) and a calm color palette (`Calm Lagoon`) for high readability. The use of CSS variables (Sass-like structure) ensures a clean, maintainable, and visually appealing codebase.
  • Functionality & Robustness:
    All hyperlinking is tested. The `selfcare.html` checklist utilizes robust JavaScript logic with Local Storage, proving a functional, interactive element that remembers the user's progress even after closing the browser.
  • Source Code Quality & Documentation:
    Code is meticulously structured with detailed section comments (e.g., `[01] CSS Variables`, `[04] Section & Card Styling`), ensuring clarity, high maintainability, and ease of review for the assessor.
